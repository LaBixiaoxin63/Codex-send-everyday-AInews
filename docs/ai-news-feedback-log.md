## 当前实现备注

- 发送前去重不再只依赖标题 key，而是增加内容指纹与同事件相似度判断。
- 相同事件的不同来源报道会优先保留真实来源名或链接更权威的条目，例如官方博客、Reuters、Bloomberg、主流科技媒体。
- 当前摘要采用早报文字版结构：标题主线、概览列表、编号详细内容、为什么看、来源和原始链接；已移除容易与标题重复的“一句话”行。
- 每周日 21:30 北京时间会发送 `本周AI大事看点`，参考产品君“一周 AI 大事”式快节奏清单：开场总结一周主线，随后列出 5-8 条模型、工具、产品、开源、算力、监管和商业化硬进展。
- 每次运行会发送两篇：`AI早报｜全球篇` 和 `AI早报｜国内篇`。
- 全球篇默认最多 8 条，国内篇默认最多 8 条；两篇分别筛选和去重，低信号内容不会为了凑条数进入日报。
- 国内篇重点覆盖 DeepSeek、通义千问、Kimi、智谱、豆包、MiniMax、百度、阿里、腾讯、华为、字节跳动等；中文媒体报道海外公司仍归入全球篇。
- 国内篇现在也会参考量子位、IT之家、InfoQ、36氪等中文直连 RSS，以及 DeepSeek/Qwen/GLM 等官方 GitHub Atom 源；Google News 不再是唯一有效信号源。
- 国内篇会优先选择 DeepSeek、Kimi、千问、通义、豆包等模型/工具/产品进展；访谈、圆桌、观点、人物对话默认不进入，除非标题本身有明确发布、版本、产品或技术事件。
- RSS 抓取会校验返回内容是否真的是 RSS/Atom；如果服务返回 HTML 搜索页，会按失败源处理，不再误判为“成功但无新闻”。
- 日志会输出成功源名称和失败源名称，例如 `Feeds ok: 量子位 RSS, IT之家 RSS`，方便判断 Google/Bing 是否实际参与。
- RSS 抓取使用单源失败隔离；全部源失败时会发送明确失败说明，避免定时任务静默失败。
- 可用 `AI_NEWS_SCOPE=domestic` 或 `AI_NEWS_SCOPE=global` 只发送单篇，默认不设置时仍发送全球篇和国内篇两篇。
- Workflow 现在有 06:53 和 07:23 北京时间两次定时触发，用第二次降低 GitHub Schedule 偶发延迟或丢失的风险。
- 真实 Google News RSS dry-run 在 2026-05-23 验证时因 `news.google.com:443` 连接超时未跑完；本地构造的重复样例已验证去重逻辑有效。
